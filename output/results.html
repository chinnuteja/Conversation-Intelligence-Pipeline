System architecture

Stage 1 — Ingest & normalize

Parse conversations and messages; split agent text at End of stream for visible reply vs product JSON (hallucination ground truth).

Python · Pydantic · JSON or MongoDB (optional)

Stage 2 — LLM-as-judge

Per-conversation structured evaluation (Azure OpenAI, json_schema response format): factual accuracy, hallucination vs catalog, policy, tone, satisfaction signals, cross-brand check.

Azure OpenAI · asyncio · concurrent evaluations

Stage 3 — Embed, cluster, discover

Embed failure text with sentence-transformers; UMAP reduction; HDBSCAN; LLM cluster labels; conservative merge + overlap pass.

sentence-transformers · UMAP · HDBSCAN

Stage 4 — Aggregate & report

Brand rollups, top failure clusters per brand, executive summary LLM pass; JSON + Markdown + Streamlit dashboard + this HTML page.

pandas · Streamlit · Plotly

Key design decisions

  • Structured JSON output — Guarantees parseable evaluations vs free-form JSON that can break downstream aggregation.
  • HDBSCAN over k-means — No fixed cluster count; handles noise points suitable for exploratory issue discovery.
  • Local embeddings — all-MiniLM-L6-v2 on CPU: fast, no extra API cost for clustering text.
  • Thin orchestration — Direct SDK calls where a single structured completion suffices; avoids heavy framework overhead for this pipeline.
